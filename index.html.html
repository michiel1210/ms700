DLP policies

Prevent sensitive data from being shared in Teams. Configured in Microsoft Purview.

  • Applies to Teams chat AND channel messages
  • Can block messages, show policy tips, notify admins
  • Requires M365 E3/E5 or compliance add-on
  • Also applies to guests in scope

Retention policies

LocationStored in
Teams channel messagesHidden Exchange mailbox + SharePoint
Teams private chatsUser Exchange mailbox hidden folder
Meeting recordingsOneDrive (non-channel) or SharePoint (channel)
Teams retention is configured SEPARATELY from Exchange. You must explicitly target Teams channel messages or Teams chats as locations in Purview.

Sensitivity labels

  • Applied to Teams, M365 Groups, SharePoint sites
  • Controls privacy (Public/Private), guest access, device access, meeting policies
  • Must be published via a label PUBLISHING POLICY - creating the label alone is not enough
  • Requires Entra ID P1

Information Barriers

Prevents specific groups from communicating (e.g. traders vs analysts). Configured in Microsoft Purview.

  • Requires Microsoft 365 E5 or E5 Compliance
  • Uses Entra ID attributes like Department to define segments
  • IB modes: Open, Owner Moderated, Implicit, Explicit
  • When applied: blocked users automatically removed from teams

External collaboration

TypeWhat it isConfigured in
External access1:1 chat and calls with other orgs. No access to your teams or channels.TAC only

Guest access requires ALL 3 enabled: (1) Entra ID External Identities, (2) M365 Admin Center group settings, (3) TAC guest access. Any one off = guests cannot join.

Governance

Group Expiry Policy

  • Expires groups after X days of inactivity
  • Owners get emails at 30, 15, 1 day before expiry
  • Not renewed = soft-deleted 30 days then permanently deleted
  • Requires Entra ID P1. Configure at Entra ID - Groups - Expiration

Naming Policy

  • Enforce prefix/suffix (e.g. GRP_ prefix)
  • Block specific words
  • Requires Entra ID P1. Global admins are EXEMPT.
  • Configure at Entra ID - Groups - Naming policy

Hands-on Labs

Use your E5 trial tenant. Each lab takes 10-20 minutes.

LAB 01 Network Planner
  1. TAC - Planning - Network planner - Add plan
  2. Add a network site, set subnet and WAN link speed (e.g. 100 Mbps)
  3. Add 50 users with Office worker persona
  4. Run the report and note bandwidth usage percentage
  5. Reduce link speed to 10 Mbps and observe the insufficiency warning
LAB 02 Guest Access (all 3 layers)
  1. Entra ID - External Identities - External collaboration settings - Allow guests
  2. M365 Admin Center - Settings - Org settings - M365 Groups - Allow guests
  3. TAC - Users - Guest access - Turn On
  4. Create a team, go to Members, add member using external email
  5. Test the guest experience from a personal email
LAB 03 Create and Assign a Meeting Policy
  1. TAC - Meetings - Meeting policies - Add
  2. Name it No External Recording
  3. Disable: Allow recording for attendees, Allow transcription
  4. Enable: Allow IP video, Screen sharing Entire screen
  5. Save and assign to your own account
  6. Schedule a test meeting and verify the policy applied
LAB 04 Group Expiration Policy
  1. Entra ID - Groups - Expiration
  2. Set group lifetime to 180 days
  3. Set admin email for groups with no owner
  4. Select All groups and save
  5. Note the Entra ID P1 licensing requirement warning
LAB 05 Group Naming Policy
  1. Entra ID - Groups - Naming policy
  2. Add prefix: GRP_
  3. Add a blocked word: Confidential
  4. Save and then create a new Team to see enforcement
LAB 06 DLP Policy for Teams
  1. Go to compliance.microsoft.com (Microsoft Purview)
  2. Data loss prevention - Policies - Create policy
  3. Template: Financial - Credit Card Number
  4. Add location: Teams chat and channel messages
  5. Action: Block message and show policy tip
  6. Test by sending a fake credit card number in Teams chat
LAB 07 PowerShell Teams Management
  1. Run: Install-Module MicrosoftTeams
  2. Connect: Connect-MicrosoftTeams
  3. List teams: Get-Team
  4. Create team: New-Team -DisplayName "Lab Team" -Visibility Private
  5. Add yourself: Add-TeamUser -GroupId [id] -User youremail -Role Owner
  6. Archive: Set-TeamArchivedState -GroupId [id] -Archived $true
LAB 08 External Access by Domain
  1. TAC - Users - External access
  2. Switch from Allow all to Allow specific domains
  3. Add an allowed domain (e.g. microsoft.com)
  4. Add a blocked domain (e.g. competitor.com)
  5. Note: this is federation for 1:1 chat, not guest access
LAB 09 CQD and Call Analytics
  1. TAC - Analytics and reports - Call quality dashboard
  2. Explore Summary Reports and note audio/video stream quality
  3. TAC - Users - select a user - Call history
  4. Click a call and review jitter, packet loss, latency details
  5. Remember: CQD = org-wide, Call Analytics = per-user
LAB 10 Fun: Set Up a Webinar
  1. Teams client - Calendar - New meeting - Webinar
  2. Add title, date, add yourself as co-presenter
  3. Enable registration form with a custom question
  4. Disable Allow attendees to unmute
  5. Publish and copy the registration link
  6. Open in private browsing, register, and join from both sides
